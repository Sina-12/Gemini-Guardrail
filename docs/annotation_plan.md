Our unannotated but preproccessed data consists of "successful" arguments (threads that changed the original poster's (OP's) mind) and "unsuccessful" arguments (threads that did not), with Gemini-generated summaries of each. Our desired annotation is then how "accurate" the summary is, labeled using the following three criteria:
 - Sentiment (bool): did the LLM correctly mark the argument thread as successful or unsuccessful at changing the OP's mind?
 - Accuracy (int): did the LLM accurately represent the arguments used in the thread? Subjectively marked, 0 = not very accurate or worse, 1 = moderately accurate, 2 = very accurate or better.
 - Succinctness (int): did the LLM capture the key points of the arguments used in the thead? Subjectively marked, 0 = captured few points, 1 = captured some points, 2 = captured most points. (This can be compared against an auto-calculated "%summarized" value, i.e. roughly how much shorter the summary is word count-wise.)

We feel that it is necessary for humans to annotate this subjective data, as our whole point is analyzing LLM summary accuracy, not LLM accuracy of evaluating LLM summaries. There are around 275 pairs of entries (224 pairs processed from an existing corpus, ~50 pairs additionally scraped from Reddit, i.e. around 550 total entries) we need to annotate, with four of us in our group. We will attempt to enlist undergraduate help for our analysis, but in the worst case scenario of nobody being available to help, each of us four will annotate half of the entries. The end goal is for each entry to be annotated at least twice, with the average Accuracy and Succinctness values taken, and conflicting Sentiment analyses ran by the entire group to decide.

This annotation process should be relatively straightforward, so the focus should be on ensuring that participants know the definitions of the criteria clearly and that they do not do too many annotations at a time to avoid fatigue. Spreading the work throughout the week should allow for it to be done comfortably. We expect nearly all of our data from the existing corpus to be annotated, quiz scheduling permitting, with some doubts about the scraped data, as it needs to go through some more preprocessing (or really, a round of pre-annotation) before it is ready to be annotated.
